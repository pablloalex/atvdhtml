Designing Web Pages with CSS

1. O que é CSS

CSS (Cascading Style Sheets) é a linguagem de estilo que controla a aparência visual dos elementos HTML no navegador — fontes, cores, espaçamentos, bordas e layout. Sem CSS, páginas ficam com aparência padrão e sem atratividade.

Estrutura de uma regra CSS

Uma regra CSS é composta por seletor e declaração. A declaração contém uma ou mais duplas de propriedade: valor.

h1 {
  font-family: Arial;
  color: darkblue;
}

2. Formas de incluir CSS no HTML

CSS externo (recomendado)

O código CSS fica em um arquivo separado (.css) e é referenciado no HTML com a tag <link>. Permite reutilização em vários documentos e manutenção centralizada.

<link href="css/styles.css" type="text/css" rel="stylesheet" />

CSS interno

As regras ficam dentro do elemento <style> no <head> do HTML. Útil para páginas únicas, mas impede reuso.

CSS inline

Estilos são aplicados diretamente no atributo style de cada elemento. Deve ser evitado, pois elimina a reusabilidade.

@import

Permite importar arquivos CSS adicionais dentro de um arquivo CSS já existente, criando um sistema modular de estilos.

3. Seletores

Seletores determinam quais elementos HTML uma regra CSS afeta.

Seletor Exemplo O que seleciona
Tipo h1 { } Todos os elementos <h1>
Classe .intro { } Elementos com class="intro"
ID #topo { } Elemento com id="topo"
Universal * { } Todos os elementos
Atributo [type="text"] { } Elementos com atributo específico
Filho direto body > ul { } Filhos diretos de <body>
Descendente body ul { } Qualquer <ul> dentro de <body>
Irmão adjacente h1 + p { } <p> imediatamente após <h1>

4. Cascata e Especificidade

Quando múltiplas regras se aplicam ao mesmo elemento, o CSS define qual prevalece:

  • Mesmo seletor: a última regra declarada vence.
  • Seletores diferentes: o mais específico vence.

A especificidade é calculada com três contadores A·B·C:

  • A — incrementa para cada seletor de ID
  • B — incrementa para classes, atributos e pseudo-classes
  • C — incrementa para seletores de tipo e pseudo-elementos
* {}           /* 0·0·0 */
h1 {}          /* 0·0·1 */
.valid {}      /* 0·1·0 */
#header {}     /* 1·0·0 */
Pseudo-classes e Pseudo-elementos

Pseudo-classes (:hover, :focus, :nth-child()) selecionam elementos em estados específicos. Pseudo-elementos (::before, ::after, ::first-letter) endereçam partes específicas de um elemento.

5. Herança

Propriedades como color e font-family são herdadas pelos elementos filhos automaticamente. Outras, como border, não são — o que evita que bordas de um pai se repliquem em todos os descendentes.

6. Cores e Formatação de Texto

Formas de definir cores

  • rgb(255, 255, 0) — valores RGB
  • #FFFF00 — hexadecimal
  • yellow — nome de cor (140+ disponíveis)
  • rgba(255, 255, 0, 0.5) — RGB com opacidade
  • hsl(60, 100%, 50%) — matiz, saturação e luminosidade

7. Listas e Tabelas

Listas

A propriedade list-style-type controla o marcador:

  • Não-ordenadas: disc, circle, square, none
  • Ordenadas: decimal, lower-alpha, upper-roman, etc.

Também é possível usar uma imagem como marcador com list-style-image: url(...), e controlar a posição com list-style-position: inside | outside.

Tabelas

Principais propriedades para estilizar tabelas:

  • border — define borda da tabela
  • border-collapse: collapse — elimina bordas duplas entre células
  • padding — espaçamento interno das células
  • tr:nth-child(odd/even) — estilo alternado por linha
  • td:first-child / td:nth-child(n) — estiliza colunas específicas

8. Sistemas de Layout

Conceitos base

Elementos HTML se dividem em block-level (empilhados verticalmente, como <div>, <p>) e inline-level (fluem horizontalmente, como <span>, <a>). A propriedade display permite alterar esse comportamento.

Tipos de posicionamento via position:

  • static — fluxo normal (padrão)
  • relative — deslocado em relação à posição normal
  • absolute — relativo ao elemento pai
  • fixed — relativo à janela do navegador
  • sticky — misto entre relative e fixed

Float Layout

A propriedade float: left | right posiciona elementos à esquerda ou direita do bloco pai, com o texto fluindo ao redor. Foi amplamente usada no passado, mas hoje existem alternativas mais adequadas.

Flexbox Layout

Introduzido no CSS3, o flexbox é um layout unidimensional (linhas ou colunas) e direção-independente. O container é ativado com display: flex.

PropriedadeAplica-se aFunção
flex-directionContainerDireção dos itens (row/column)
justify-contentContainerDistribuição no eixo principal
align-itemsContainerAlinhamento no eixo cruzado
flex-growItemQuanto o item pode crescer
flex-shrinkItemQuanto o item pode encolher
flexItemShorthand para grow/shrink/basis
orderItemOrdem de exibição

Grid Layout

O grid é um layout bidimensional (linhas e colunas simultâneas), ideal para layouts complexos. Ativado com display: grid.

form {
  display: grid;
  grid-template-columns: 1fr 2fr;
  grid-gap: 16px;
}

Itens são posicionados com grid-column: 1 / 2 e grid-row. Também é possível nomear áreas com grid-template-areas.

9. Pontos-Chave

  • CSS controla a aparência de elementos HTML via regras (seletor + declaração).
  • CSS pode ser incluído de forma externa (recomendado), interna ou inline.
  • Seletores permitem selecionar elementos por tipo, classe, ID, atributo, hierarquia, etc.
  • A especificidade determina qual regra prevalece quando há conflito.
  • Propriedades como color e font-family são herdadas pelos filhos.
  • CSS oferece três sistemas de layout modernos: Float, Flexbox e Grid.
  • Flexbox é ideal para layouts unidimensionais; Grid é mais poderoso para layouts complexos.
